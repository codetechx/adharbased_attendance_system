4Database Schema & Relationships
Key design decisions: Vendors link to Companies through the company_vendors pivot — status drives whether workers appear in attendance. Workers have fingerprint_template stored AES-encrypted via Laravel's encrypt()/decrypt(). Aadhaar number is never stored in full — only the masked last-4-digits string.

7Fingerprint Attendance — Technical Flow
frontend/src/pages/attendance/AttendanceMark.jsx
const SGIBIOSRV = "https://localhost:8443"; // Step 1: capture from local SGIBIOSRV (bypass CORS via text/plain) async function sgiCapture() { const res = await fetch(`${SGIBIOSRV}/SGIFPCapture`, { method: "POST", headers: { "Content-Type": "text/plain;charset=UTF-8" }, body: "Timeout=10000&Quality=50&templateFormat=ISO", }); return res.json(); } // Step 3: 1:N match via SGIBIOSRV (score 0–200) async function sgiMatchScore(t1, t2) { const body = `template1=${encodeURIComponent(t1)}&template2=${encodeURIComponent(t2)}&templateFormat=ISO`; const res = await fetch(`${SGIBIOSRV}/SGIMatchScore`, { method: "POST", headers: { "Content-Type": "text/plain;charset=UTF-8" }, body, }); return res.json(); } // All workers matched in parallel — best score wins const results = await Promise.all( workers.map(async w => { const v = await sgiMatchScore(probeTemplate, w.template); return { worker: w, score: v.ErrorCode === 0 ? v.MatchingScore : 0 }; }) ); const best = results.reduce((a, b) => a.score > b.score ? a : b); // MATCH_THRESHOLD = 40 (out of 200)
backend/app/Http/Controllers/AttendanceController.php — workerTemplates()
public function workerTemplates(Request $request): JsonResponse { // Resolve company — gate user implicit, super_admin must pass param $companyId = $request->input('company_id') ?? $user->company_id; // Get approved vendor IDs for this company from pivot $approvedVendorIds = DB::table('company_vendors') ->where('company_id', $companyId) ->where('status', 'approved') ->pluck('vendor_id'); // Get active workers with enrolled fingerprints $workers = Worker::with('vendor') ->whereIn('vendor_id', $approvedVendorIds) ->whereNotNull('fingerprint_template') ->where('status', Worker::STATUS_ACTIVE) ->get(); // Decrypt each template before sending to browser return response()->json($workers->map(fn($w) => [ 'worker_id' => $w->id, 'template' => decrypt($w->fingerprint_template), 'pending_type' => $this->getPendingType($w->id, $companyId), // ...other fields ])); }
⚠ SGIBIOSRV cert: Browsers block direct HTTPS to localhost unless the user visits https://localhost:8443 and accepts the self-signed cert. The text/plain Content-Type avoids CORS preflight — making it a "simple request" that bypasses the CORS check entirely.

9Pushing Backend Changes (Critical)
The backend PHP source is baked into the Docker image — it is NOT volume-mounted. Saving a .php file locally does NOT update the running container. You must use docker cp.

Standard Workflow for Any Backend Change

1

Edit the PHP file locally

Make your changes in backend/app/... or backend/routes/api.php as normal.

2

Copy changed file into the running container

# Controller docker cp backend/app/Http/Controllers/WorkerController.php \ ams_backend:/var/www/html/app/Http/Controllers/WorkerController.php # Model docker cp backend/app/Models/Worker.php \ ams_backend:/var/www/html/app/Models/Worker.php # Routes docker cp backend/routes/api.php \ ams_backend:/var/www/html/routes/api.php
3

Clear Laravel's route + config cache

docker exec ams_backend php artisan optimize:clear
4

Only rebuild the image when you add packages (composer)

# Only needed when you run `composer require something-new` docker compose build backend docker compose up -d
Frontend changes are instant — the ./frontend:/app volume is live. If Vite HMR doesn't pick up a change, run docker compose restart frontend and hard-refresh the browser.

11Debugging Guide

Common Errors & Root Causes

ErrorCause & Fix
401 Unauthenticated Token expired or missing. Check localStorage has ams_token. Token expires after 7 days.
403 Insufficient permissions Role middleware blocking route. Check routes/api.php middleware group for the endpoint.
422 Unprocessable Laravel validation failed. Response body has errors object with field details.
500 Server Error PHP exception. Check: docker logs ams_backend --tail=50
Frontend change not showing Vite HMR missed it. Run docker compose restart frontend, then hard-refresh (Ctrl+Shift+R).
Backend change not reflecting Forgot docker cp. PHP is baked in image. Always cp + optimize:clear.
SQLSTATE Column not found Used wherePivot() inside whereHas() — use where('table.column', value) instead.
Fingerprint ErrorCode 10004 SGIBIOSRV timeout — user didn't place finger within 10 seconds. Not a code bug.
Fingerprint ErrorCode 53 Device not found. Check USB. Try different port. Restart SGIBIOSRV service.
CORS error on fingerprint Browser hasn't accepted the self-signed cert. Visit https://localhost:8443 and click Proceed.

12Hosting & Deployment
Shared hosting won't work. The app requires Docker, Redis, Python, and PHP-FPM with custom nginx config — none of which are available on shared hosting (cPanel/Hostgator etc.). You need a VPS.

Server Setup (One Time)

# SSH into your VPS (Ubuntu 22.04 recommended) ssh root@your-server-ip # Install Docker curl -fsSL https://get.docker.com | sh # Clone project git clone https://github.com/yourrepo/attendance_system.git cd attendance_system # Configure environment cp .env.example .env nano .env # set APP_KEY, DB_PASSWORD, etc. # Generate app key docker compose run --rm backend php artisan key:generate # Start all services docker compose up -d # Run migrations + seed docker exec ams_backend php artisan migrate --seed

Add HTTPS (Free SSL via Let's Encrypt)

# Install certbot on the host (Ubuntu) apt install certbot python3-certbot-nginx # Stop nginx container temporarily (needs port 80 free) docker compose stop nginx # Issue certificate certbot certonly --standalone -d yourdomain.com # Update nginx/conf.d/default.conf to add SSL listener # Then restart docker compose start nginx
Production checklist: Set APP_ENV=production, APP_DEBUG=false, use strong DB/Redis passwords, restrict port 3306 from public access, set up automated MySQL backups.
