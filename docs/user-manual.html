1 What is AMS?

AMS (Attendance Management System) is a digital system that tracks when contract workers arrive and leave a company site — using fingerprint scanning, so there is no manual register or paper-based attendance.


Old WayWith AMS
📝 Paper register — can be faked✅ Fingerprint — cannot be faked
❌ Manual entry — errors, delays✅ Auto-recorded with exact timestamp
❌ No visibility for vendor✅ Vendor sees their workers' attendance live
❌ Hard to audit or dispute✅ Full audit trail, every action logged
2 Who Uses AMS? (Roles)
RoleWho is this person?What they do in AMS
Super Admin System owner / IT team Creates companies, vendors, all user accounts; full access to everything
Company Admin HR Manager / Site Manager Approves vendors, creates site users (Gate/Department/Other), views attendance
Gate / Dept User Security guard, HR, or department staff Scans fingerprints to mark workers IN/OUT — tagged with their gate or department location
Vendor Admin Contractor company manager Registers workers, requests company access, views worker attendance
Vendor Operator Field HR at vendor company Registers workers only (no access to attendance or approvals)

4 Getting Started — Logging In
1

Open the Application

Open Chrome or Edge browser and go to the URL provided by your IT team (e.g. http://192.168.1.10 or your company's domain).

2

Enter Your Credentials

Type the email address and password shared by your Super Admin or Company Admin. Passwords are case-sensitive.

3

View Your Dashboard

After login you see your Dashboard — today's attendance summary and key numbers. The left sidebar shows options available to your role.

⚠ Forgot your password? Contact your Super Admin or Company Admin. There is no self-service password reset — they will set a new one for you.
5 Super Admin Guide

5.1 — Create a Company

Sidebar → Administration → Companies → click Add Company

1

Fill in Company Details

Company Name, Code (short ID), Address, City, State, PIN, Contact Person, Email, Phone, GST Number (optional).

2

Create Company Admin Login

Enter Admin Name and Login Email. Click the 🔄 icon to auto-generate a strong password. Click 📋 to copy it. This password is shown only once — copy it before saving.

3

Click Create Company & Share Credentials

Send the login email and password to the Company Admin via WhatsApp or email. Ask them to confirm they can log in.

5.2 — Create a Vendor

Sidebar → Administration → Vendors → click Add Vendor

Same process as Company — fill in vendor details, create Vendor Admin login (name + email + generated password), copy and share credentials with the Vendor Admin.

ℹ After creating both Company and Vendor: The Vendor Admin must log in and send a Company Access Request. The Company Admin must then approve it. Only after approval can attendance be marked.

5.3 — Manage All Users

Sidebar → Administration → Users

Create any user type: Company Admin, Gate User, Vendor Admin, Vendor Operator. Select the role, then assign to the correct Company or Vendor. Toggle Active/Inactive to enable or disable a login.

5.4 — Edit or Deactivate a Company / Vendor

Each card in the Companies or Vendors list has a ⋮ three-dot menu in the top-right corner. Click it to Edit details or Activate / Deactivate the entire account.

6 Company Admin Guide

6.1 — Approve a Vendor

Sidebar → Administration → Vendor Approvals

1

Click the Pending tab

New vendor requests appear here with a count badge (e.g. Pending ①).

2

Click Approve

The vendor's workers are now available for fingerprint attendance at your gate — no further steps needed.

3

Or — Reject with reason

Type a reason in the text box, then click Reject. The vendor can see the reason and send a new request after fixing the issue.

💡 Suspend vs Reject: Use Suspend (on the Approved tab) to temporarily block a vendor without deleting the relationship. You can re-approve later. Use Reject to formally decline a request.

6.2 — Create Site Users (Gate / Department / Other)

Sidebar → Administration → Users → click Add User

Company Admin can create three types of site users. Each type marks attendance tagged with their specific location — so you always know where each scan happened.

Login TypeWho uses itLocation Name example
Gate Security guard at a main entry / exit point Gate 1, East Gate, Main Gate, Back Entry
Department HR or admin inside a building or department Finance Dept, IT Block, Production Floor, HR Office
Other / Checkpoint Any other scanning point on site Canteen, Parking Area, Building B, Warehouse Entry
1

Enter name and email

Use a descriptive name like "Main Gate Guard" or "HR Dept – Finance" so it is immediately clear which point this account covers.

2

Select Login Type and Location Name

Choose Gate, Department, or Other. Then type the specific location name (e.g. "Gate 1"). All attendance marked by this user will be tagged with that location in the attendance log.

3

Generate password and save

Click 🔄 to auto-generate a strong password. Click 📋 to copy. This password is shown only once — copy it before saving. Share with the person who will use this account.

⚠ One account per scanning point: Create a separate user account for each gate, department, or checkpoint. This ensures attendance records correctly show which location each scan came from.
💡 Example setup for a site with 2 gates + 1 department:
• Account 1 — Gate → "Main Gate"
• Account 2 — Gate → "Back Entry"
• Account 3 — Department → "HR Office"

7.2 — Register a Worker (Step by Step)

Sidebar → Workers → Register Worker

1

Step 1 — Aadhaar Upload

Click Open UIDAI Portal. The worker logs in on the government site, completes OTP, and downloads their masked Aadhaar PDF. Upload that PDF here.

PDF Password format: First 4 letters of name (UPPERCASE) + birth year.
Example: Name "Narendra", born 1955 → password is NARE1955
2

Step 2 — Review Details

The form auto-fills with name, DOB, gender, address from the PDF. Check for any errors and correct if needed. Click Next.

3

Step 3 — Fingerprint Enrollment

Connect the SecuGen scanner. Ask the worker to place their right index finger firmly on the scanner. Click Scan Fingerprint. The scanner light blinks — keep finger still for 2 seconds. Green tick = success.

💡 Scanner not available today? Click "Skip for now" — the worker is saved as "Pending". Enroll fingerprint later from the Workers list. The worker cannot mark attendance until fingerprint is enrolled.
4

Step 4 — Confirm & Save

Review all details. Click Register Worker. Worker status becomes Active — they can now mark attendance at any approved company site.

7.5 — Deploy Workers to a Company

Sidebar → Workers → Deploy Workers

After a company approves your vendor, deploy specific workers with a start and end date. The deployment list has Current / Previous / All tabs.

💡 Cancelling a deployment: You can cancel an active deployment even after attendance has been marked — as long as the worker is not currently checked IN. If the worker is inside, have them check OUT first, then cancel.

8 Gate / Department User Guide

This guide applies to Gate, Department, and Other/Checkpoint users — all types mark attendance the same way. Every scan is automatically tagged with your Login Type and Location Name as configured by your Company Admin (e.g. "Gate 1", "HR Office", "Canteen").

8.1 — One-Time Setup on the Gate / Dept PC

⚠ Do this once per PC before first use: The fingerprint scanner needs browser permission to connect.
1

Connect the scanner

Plug the SecuGen fingerprint device into any USB port. The LED light should come on.

2

Accept the scanner certificate

In Chrome/Edge, open https://localhost:8443 — you will see a security warning:

⚠ Your connection is not private

Click Advanced at the bottom → then click "Proceed to localhost (unsafe)"

You will see a plain white page — that's normal. This just enables the scanner to work.

3

Log in to AMS and go to Mark Attendance

Sidebar → Attendance → Mark Attendance. You are ready to scan.

8.2 — Troubleshooting

ProblemWhat to do
No finger detectedWorker must place finger immediately after clicking Scan — do not wait. Try again quickly.
No fingerprint match foundClean the worker's finger and sensor. Press harder, keep still. If persists, ask vendor admin to re-enroll fingerprint.
Cannot reach scanner / connection errorCheck USB cable. Open https://localhost:8443 in browser and accept the certificate again.
Scanner light doesn't blinkDevice not connected — try different USB port. Restart the SGIBIOSRV service.
No workers found for scanningNo vendor is approved for your company yet. Company Admin must go to Vendor Approvals and approve a vendor.
Worker shown as wrong IN/OUTSystem tracks last action automatically. If IN was not marked, this OUT won't work — contact Company Admin to manually correct.

9.1 — Register a Worker as Operator

Sidebar → Workers → Register Worker

The registration process is identical to the Vendor Admin flow — refer to Section 7.2 for the full step-by-step guide (Aadhaar upload → details → fingerprint → confirm).

💡 Note: Workers registered by an Operator are automatically assigned to the same vendor. The Vendor Admin will see them in the workers list and can deploy them to companies.

9.2 — Viewing Workers and Attendance

Operators can view the full Workers list (All / Current / Previous tabs) and click any row to see that worker's attendance analytics. The Attendance Log is also available in read-only mode — filtering by date, name, and deployment tab works the same way.

⚠ Operators cannot mark attendance — that is the Gate User's role. If you need to mark attendance, contact your Company Admin to set up a Gate User account at the site.
10 Onboarding Checklist

🟣 Super Admin — New Company Setup

  • Create Company in AMS (Administration → Companies → Add Company)
  • Generate and copy Company Admin password
  • Share login credentials with Company Admin (email + password)
  • Confirm Company Admin can log in successfully

🟣 Super Admin — New Vendor Setup

  • Create Vendor in AMS (Administration → Vendors → Add Vendor)
  • Generate and copy Vendor Admin password
  • Share login credentials with Vendor Admin (email + password)
  • Confirm Vendor Admin can log in successfully

🟠 Vendor Admin — Getting Started

  • Log in to AMS with provided credentials
  • Go to Company Access → find your client company → Request Access
  • Wait for company approval (follow up if needed)
  • Register at least one worker with Aadhaar PDF
  • Enroll fingerprint (scanner must be connected)
  • Confirm worker status shows Active
  • Go to Deploy Workers → assign worker to the company with a date range

🔵 Company Admin — Getting Started

  • Log in to AMS with provided credentials
  • Go to Vendor Approvals → Pending tab → Approve vendors
  • Go to Users → Add User — select Login Type (Gate / Department / Other) and enter the Location Name
  • Create one account per scanning point (gate, department, checkpoint)
  • Share credentials with the person at each scanning point
  • Confirm each user can log in and reach Mark Attendance

🟡 Vendor Operator — Getting Started

  • Log in to AMS with credentials provided by your Vendor Admin
  • Go to Workers → Register Worker to add new workers
  • Upload Aadhaar PDF, review auto-filled details, enroll fingerprint
  • Confirm worker status shows Active after fingerprint enrollment

🩵 Gate / Department User — PC Setup (One Time)

  • Plug SecuGen scanner into USB on gate PC
  • Open browser → go to https://localhost:8443 → Accept certificate
  • Log in to AMS with gate user credentials
  • Go to Mark Attendance
  • Test with one known worker fingerprint to confirm scanner works
11 Frequently Asked Questions
A worker's fingerprint is not being recognised. What do I do?
Clean the worker's finger and the scanner sensor with a dry cloth. Ask them to press firmly and keep the finger completely still. If it still fails after 3 tries, ask the vendor admin to go to the Workers list, open the worker's profile, and re-enroll the fingerprint.
The gate scanner is not working / shows a connection error.
1. Check that the SecuGen USB scanner is properly plugged in.
2. Open a new browser tab and go to https://localhost:8443. If you see a security warning, click Advanced → Proceed to localhost.
3. Refresh the Mark Attendance page and try again.
A vendor's workers are not showing up for fingerprint scanning.
This means the vendor has not been approved for your company yet. The Company Admin must go to Vendor Approvals and approve the vendor. Once approved, their workers are immediately available at the gate — no further steps needed.
How do I add more workers under our vendor?
The Vendor Admin or Vendor Operator can register new workers at any time via Register Worker. Once registered and fingerprint enrolled, they are immediately available at all companies that have approved your vendor.
Can a worker be at two companies on the same day?
Yes. If your vendor is approved at multiple companies, a worker can mark IN/OUT at any of them on the same day. Each attendance record shows which company it was marked at, and the timestamp.
What if a worker forgot to mark OUT at the end of the day?
The Company Admin can see all such cases in Attendance → Exceptions. Corrections can be made manually from the Attendance Log page. Only Company Admin and Super Admin can do manual corrections — Gate Users cannot.
How do I change a gate user's password?
The Company Admin goes to Gate Users, clicks the pencil (edit) icon next to the user, enters a new password, and clicks Save. Share the new password with the gate user directly.
I cannot log in — it says incorrect email or password.
Make sure you are using the exact email address and password provided by your admin. Passwords are case-sensitive. If you are still unable to log in, contact your Super Admin or Company Admin to reset your password.
We want to stop a vendor from working at our site. What do we do?
The Company Admin goes to Vendor Approvals → Approved tab, then clicks Suspend next to the vendor. Their workers' fingerprints will immediately stop working at your gate. You can re-approve them at any time without them needing to send a new request.
The Aadhaar PDF is not uploading / extraction failed.
Make sure the PDF is the official masked Aadhaar downloaded from the UIDAI portal (myaadhaar.uidai.gov.in). Check that you are entering the correct password: first 4 letters of the worker's first name in UPPERCASE + the 4-digit birth year (e.g. NARE1955). If the issue persists, contact your IT administrator.
How do I view a worker's full attendance history?
Click any worker row in the Workers list or in the Attendance Log to open the Worker Detail analytics page. It shows total days worked, average hours, a monthly breakdown table, deployment history, and recent attendance — all scoped to your company. Vendor Admins can also use the company dropdown to switch between companies and compare history across sites.
Can I download a worker's Aadhaar or ID document?
Yes. The Workers list has an "ID Document" column with a Download button for each worker's primary document. The same download is available in the worker's edit page (Step 0). Vendor users can always download their workers' documents. Company users can download if the worker has been deployed to or has attended their company.
I cancelled a worker's deployment — why do they still appear in Previous Workers?
This is correct behaviour. Once a worker has marked attendance at your company, they will always appear in the Previous Workers tab — even if the deployment is later cancelled. This ensures a complete attendance history is maintained. The Previous tab shows all workers who ever worked at your site.
Can I cancel a deployment after attendance has been marked?
Yes — as long as the worker is not currently checked IN. If the worker's most recent action is an OUT (or they haven't attended yet), the Cancel button will work. If the worker is currently inside (last action was IN), have them check OUT first at the gate, then you can cancel the deployment.
